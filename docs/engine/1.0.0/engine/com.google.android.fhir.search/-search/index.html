Search

class Search(val type: ResourceType, var count: Int? = null, var from: Int? = null) : BaseSearch

Provides a fluent API for building FHIR search queries within the Android FHIR Engine, supporting standard FHIR search parameters. This Domain Specific Language (DSL) offers a convenient and expressive way to search for resources based on various criteria, including:

  • Resource Type: Specify the type of resource to search for (e.g., Patient, Observation).

  • Search Parameters: Utilize FHIR search parameters to define the search criteria (e.g., Patient.GIVEN, Observation.CODE).

  • Filter Values: Provide specific values to filter the search results (e.g., "Alice" for Patient.GIVEN).

  • Modifiers: Apply modifiers to string-based search parameters to refine the search criteria (e.g., StringFilterModifier.CONTAINS).

  • Sorting: Order search results based on specified parameters and sort direction (e.g., Patient.FAMILY in ascending order).

  • Result Modification: Limit the number of returned results and specify the starting index.

  • Reverse Chaining: Search for resources based on other resources that reference them using the has function.

  • Include and RevInclude: Include additional referenced resources in the search results using the include and revInclude functions.

Example Usage:

// Search for all Patient resources with the given name of exactly "Kiran"
val patients = fhirEngine.search<Patient> {
 filter(Patient.GIVEN, {
   value = "Kiran"
   modifier = StringFilterModifier.MATCHES_EXACTLY
 })
}

// Search for all Patients where they are the subject of an Observation with the code
// "http://snomed.info/sct|386661006" or "http://loinc.org|45701-0" (different codes for fever).
val patients = fhirEngine.search<Patient> {
 has<Observation>(Observation.SUBJECT) {
   filter(
     Observation.CODE,
     {
       of(Coding("http://snomed.info/sct", "386661006", "Fever" ))
     }, {
       of(Coding("http://loinc.org", "45701-0", "Fever"))
     }
   )
 }
}
